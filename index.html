What are they?


First, we have to understand what Git and VS Code is, so let's start with a quick breakdown of what each is and why they are important.

Understanding Git

What is it?


The most popular version control system (VCS), free and open source. This means it manages the changes to source code.


What's special about it?


Through Git's branching model, you can easily create, merge, and delete multiple branches independent of each other, local or remote.

This allows for multiple workflows on a single project that can be implemented seamlessly. Additionally, its distributed nature means that the number of workflows can be unlimited.

Understanding VS Code

What is it?


A lightweight source code editor that runs on your desktop. It has built-in support for JavaScript, typeScript and Node.js, and has a rich library of extensions for other languages and tools.


What's special about it?


Along with an intuitive UI, VS code provides code completion tools, enahnced editing capability, easy navigation within code, and version control support through Git integration.

Clone your repository in VS Code

What is a Repository?


explain what it is


🏁

To start using VS Code and Git together, create a new repository in Github, then publish it. Afterwards, you can clone it in VS Code.


How to clone a repsository in VS

  1. Get the URL for the repository from Github.
  2. Open VS Code and select "clone git repository" from the welcom screen. If you are already working in VS, you can go to the 'source control' panel and open more options to select "clone".
  3. Enter the URL for your remote repository and proceed. This will create a copy of the repository on your local system.
💡

It will prompt you to set up a working directory for your repository. You should make a path that is intuitive to the project you are working on.

Branching with VS Code

What is Branching?


Branching is basically how you separate spaces to work on a single project. You can create a branch from an existing branch, create a new branch entirely, or create a detached branch. Each of these options have their own purpose, but the main idea is branching isolates changes you make. This keeps the integrity of your main branch together, and also manages what spaces people are working in. If you think about feature development, this allows a team to have individual branches for specific features, all of which can be created from the same main branch, and experiment before implementing them into the main branch.


This is also the same for bug fixes, as developers can assess and experiment with solutions or breaking in another branch so the in-progress edits do not affect the main branch

Commit changes to File(s)

🚨

You have to make sure git is configured on your local repository first. This only needs to be done once for each cloned repository.

  1. Open the terminal in VS Code
  2. Using your Github username and email, in terminal:
    1. Enter git config user.name "[your username]"
    2. Enter git config user.email "[your email]"

How to Commit in VS

  1. Create whatever files or changes to a file that you want.
  2. Hit “stage changes” while you hover over the file tab (its the plus sign)
  3. Add a brief message explaining what your changes were, then hit “commit”

How to Push Changes in VS


  1. Navigate to "source control" while in VS Code
  2. Hit "sync changes"
💡

Using “sync changes” will both pull and push from the remote repository. This is an easy way to keep your local repository current with the project. If you do not want to use this method, you can also use the ellipses in the side panel of the “source control” menu. This opens a list of options (push, pull, clone, fetch) to choose from.

Pull Requests


My thoughts on Pull Requests


I start this with 'my thoughts' because pull requests can be a bit confusing to understand their place in development. From my understanding, means that you are submitting some code that you have created in a separate branch from the main(default). This is meant to either garner feedback on your code or to try and have your code contributed to a project that you do not own. This is great for collaboration and revisement because it opens up an environment for discussion, one where people can give feedback and build on your work. It also is a safegaurd for version control and file integrity, as pull requests are a prerequisite to implementation.


💡

To create pull requests in VS Code, you need to install the extension GitHub Pull Requests and Issues.
