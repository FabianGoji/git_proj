We have to understand what Git and VS Code is, so let's start with a quick breakdown of what each is and why they are important.

Understanding Git

What is it?


Git is a version control system that developers use to manage and track changes in their code. It allows multiple people to work on the same project simultaneously without interfering with each other, by creating and merging branches and handling conflicts. Git stores a history of all changes made to the project, enabling developers to revert to previous versions if necessary and to see who made which changes and when. This makes it an essential tool for collaborative coding and managing complex projects.

Understanding VS Code

What is it?


Visual Studio Code (VS Code) is a powerful and versatile code editor used by developers for writing, editing, and debugging code. It supports a wide range of programming languages and comes with features like syntax highlighting, code completion, and integrated Git control, making coding more efficient and less error-prone. VS Code also offers a vast library of extensions, allowing developers to customize their environment with additional tools and features to suit their specific needs, enhancing productivity in software development.

Using Git and VS Code


The purpose here is to show some basics about configuring Git to your repositories, and using git commands within VS Code. I'll introduce a topic and go over the steps to do it, and explain the importance in a nutshell. So let's start at the beginning.

Clone your repository in VS Code

What is a Repository?


A repository on GitHub is like a special folder for a project. It's where all the files, documents, and information related to that project are kept. Think of it as a digital library bookshelf, where each book (or repository) holds all the pages (files) for one topic (project). On GitHub, people can work together on these projects, like adding new pages or editing existing ones, and everyone can see the changes. It's a way to organize, share, and collaborate on projects online, even if you're not a tech expert.


🏁

To start using VS Code and Git together, create a new repository in Github, then publish it. Afterwards, you can clone it in VS Code.


How to clone a repsository in VS Code

  1. Get the URL for the repository from Github.
  2. Open VS Code and select "clone git repository" from the welcom screen. If you are already working in VS, you can go to the 'source control' panel and open more options to select "clone".
  3. Enter the URL for your remote repository and proceed. This will create a copy of the repository on your local system.
💡

It will prompt you to set up a working directory for your repository. You should make a path that is intuitive to the project you are working on.

Branching with VS Code

What is Branching?


Branching is basically how you separate spaces to work on a single project. You can create a branch from an existing branch, create a new branch entirely, or create a detached branch. Each of these options have their own purpose, but the main idea is branching isolates changes you make. This keeps the integrity of your main branch together, and also manages what spaces people are working in. If you think about feature development, this allows a team to have individual branches for specific features, all of which can be created from the same main branch, and experiment before implementing them into the main branch.


This is also the same for bug fixes, as developers can assess and experiment with solutions or breaking in another branch so the in-progress edits do not affect the main branch

Commit changes to File(s)

🚨

You have to make sure git is configured on your local repository first. This only needs to be done once for each cloned repository.

  1. Open the terminal in VS Code
  2. Using your Github username and email, in terminal:
    1. Enter git config user.name "[your username]"
    2. Enter git config user.email "[your email]"

What are Commits?


Think of a commit on GitHub as a bookmark in a notebook. When you make changes to your project (like writing or editing pages in the notebook), a commit marks exactly what changes you made at that moment. It's like saying, "Here's what I added or changed on this page today." This bookmark helps you and others see the history of changes made over time. It's a way of keeping track of all the small or big updates you make, like a diary entry for your project, making it easy to understand how the project has evolved.


How to Commit in VS Code

  1. Create whatever files or changes to a file that you want.
  2. Hit “stage changes” while you hover over the file tab (its the plus sign)
  3. Add a brief message explaining what your changes were, then hit “commit”

Push Changes to Remote Repository, Github

What is a Push?


Imagine you've made a bunch of changes in a scrapbook at home (this is like editing files in a local repository on your computer). "Pushing" these changes to a remote repository, like GitHub, is like sending photocopies of your updated scrapbook pages to a public library. This way, your updated pages are available in the library for others to see and use (the GitHub repository). So, pushing is essentially the act of sharing your local updates with the wider world on the internet, making your latest work accessible to others and ensuring that the public library (GitHub) has the most current version of your project.


This might sound similar to commits, but they are still very different. If commits note what the changes are on your local repository, then pushing those changes is basically sending the commit(s) to the remote repository. The message you leave in a commit is even more necessary here since other collaborators need to know about the changes you made.


How to Push Changes in VS Code


  1. Navigate to "source control" while in VS Code
  2. Hit "sync changes"
💡

Using “sync changes” will both pull and push from the remote repository. This is an easy way to keep your local repository current with the project. If you do not want to use this method, you can also use the ellipses in the side panel of the “source control” menu. This opens a list of options (push, pull, clone, fetch) to choose from.

Pull Requests

My thoughts on Pull Requests


A pull request on GitHub is like putting a suggestion box in a community center. Imagine you've made some great changes or improvements to a project you found at the center (like a shared recipe book). Instead of just replacing the original, you put your suggestions (your updated pages) in the suggestion box. This is your pull request. It's a way of saying, "Hey, I made some changes I think are good, can we add them to the original project?" The community (or project owners) can then review your suggestions, discuss them, and decide whether to incorporate them into the main project. It's a collaborative and respectful way of contributing improvements to a shared project.


💡

To create pull requests in VS Code, you need to install the extension GitHub Pull Requests and Issues.


Github Pages

What is Github Pages?


GitHub Pages is a service provided by GitHub that allows you to host a website directly from your GitHub repository, making it an excellent choice for deploying a website. It's especially beneficial because it's free, easy to set up, and integrates seamlessly with Git version control. This means you can update your website simply by pushing updates to your repository. GitHub Pages is ideal for personal, project, or documentation websites, as it supports static site generators like Jekyll for more complex projects. Its simplicity and integration with GitHub's ecosystem make it a popular choice for developers looking to quickly and efficiently publish their work online.
